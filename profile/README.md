# 포도밭학원 (Vineyard Edu)

포도밭학원은 대치동에서 초등·중등 학생을 대상으로 20여년간 국어·논술 수업을 운영해 온 오프라인 교육 기관입니다.

그 과정에서 **좋은 수업만으로는 교육이 완성되지 않는다는 한계**를 반복해서 마주했습니다.
수업, 평가, 피드백, 그리고 운영이 서로 단절된 구조에서는 아무리 밀도 높은 수업이라도 축적되지 않습니다.

이 GitHub Organization은 포도밭학원이 교육 현장에서 쌓아 온 문제의식과 경험을 바탕으로,
**교육을 하나의 서비스이자 시스템으로 구조화하기 위한 기술적 실험 공간**입니다.

---

## 우리가 다루는 문제

교육 현장에는 다음과 같은 단절이 존재합니다.

- 수업은 진행되지만, 평가와 연결되지 않는 **구조**
- 평가는 존재하지만, 다음 학습으로 환원되지 않는 **결과**
- 데이터는 쌓이지만, 의사결정에 활용되지 않는 **기록**

포도밭은 이러한 문제를 웹과 AI, 그리고 시스템 설계를 통해 풀어가고자 합니다.

---

## 우리가 다루는 영역

포도밭 Organization은 개별 기능이 아닌, **교육 서비스의 역할 단위**를 중심으로 프로젝트를 구성합니다.

| 영역                              | 설명                                                         |
| --------------------------------- | ------------------------------------------------------------ |
| **교육 서비스의 외부 인터페이스** | 학원의 정체성과 교육 철학을 전달하는 웹 서비스               |
| **운영 시스템**                   | 학생·강사·수업·평가·결제를 하나의 흐름으로 관리하는 백오피스 |
| **학습과 평가**                   | AI 기반 자동 채점, 문제 출제, 피드백 구조 설계               |
| **학습 도구 실험**                | 게임화된 학습, 개별화 학습 도구, 사고 확장을 위한 인터랙션   |

## 현재 프로젝트

| 프로젝트               | 상태    |
| ---------------------- | ------- |
| 홈페이지               | 운영 중 |
| 학생 관리 ERP 백오피스 | 운영 중 |
| AI 자동 채점 시스템    | 운영 중 |
| AI 문제 출제 시스템    | 개발 중 |

각 프로젝트는 독립적으로 존재하지만, 궁극적으로는 하나의 학습 데이터 흐름 안에서 연결되는 것을 목표로 합니다.

---

## 앞으로의 방향

- 학습 개념을 체화하기 위한 **학습용 게임**
- AI를 활용한 **개별화 학습 도구**
- 평가 결과를 누적·비교·해석하는 **분석 시스템**
- 학습 → 평가 → 피드백 → 재학습으로 이어지는 **순환 구조의 고도화**

---

## 공개 범위에 대하여

본 Organization에 포함된 대부분의 프로젝트는 실제 학생 데이터와 운영 시스템을 포함하고 있어 비공개로 관리됩니다.

---

## 우리가 중요하게 생각하는 것

- 교육은 단기 성과보다 **누적되는 경험**이다.
- 평가는 선별이 아니라 **다음 학습을 설계하기 위한 도구**다.
- AI는 교사를 대체하지 않고, **교사의 판단을 확장**한다.
- 기술은 교육을 빠르게가 아니라 **지속 가능하게** 만든다.
